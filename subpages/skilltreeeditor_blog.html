Future-Proofing Unity: Tools for Sustainable, Scalable Development

Why Sustainable Tooling Matters in Unity Projects

Game development can be chaotic, especially as deadlines approach and crunch time looms. There are many ways to manage this chaosâ€”some preventative, some reactive. Tooling is a key preventative measure that can help keep projects on track and teams productive.

Let's start by identifying common problems that arise in growing Unity projects :


  • Lengthy end-to-end processes
  • Difficulty finding related project files
  • Inconsistent assets and project setup
  • Unclear or unenforced guidelines
  • Wasting time searching through project files

    • Now that we've identified our problems; we can figure out our solutions.

  • Automate repetitive steps
  • using a dedicated asset editor / finder
  • creating Asset validators and Fixers

    • Just through breaking down problems and presenting simple solutions you can reduce the amount of error and debt in your project and in turn dramatically improve the way your project scales.

    Imagine you had thousands upon thousands of files, immediately there's an issue here; Searching and finding for what you want becomes a problem in and of itself and creates this friction between the actual important work that needs to be done and simply navigating the project.

    This is a core issue as this becomes a hindrance to every member of the team. As the project grows the things that were previously possible are simply not sustainable anymore. This is why assistive tooling is vital to having a scalable and sustainable project, as they help mitigate one of the core struggles of a large project.
Principles for Scalable, Sustainable Unity Tooling

These principles will help you make the right decisions when planning tooling. They focus on assistive tooling rather than content tooling as that already has a given purpose


  • A tool must solve a problem: Tools should solve a purpose, if it has no valid purpose then it shouldn't be a thing; The last thing you want is to add additional technical debt to your project.

  • Enable Quality and Efficiency: Tools should allow for faster iteration, higher quality output, or the ability to handle more work. If a tool doesn't provide a clear benefit in one of these areas, reconsider its value. Focus on measurable improvements to workflow or results.

  • Simplify Complex Content Pipelines: Good tools break down complicated processes into manageable steps, making it easier for teams to collaborate and maintain consistency.

  • Reduce Interdisciplinary Dependencies: Tools should minimize bottlenecks between disciplines, allowing artists, designers, and programmers to work simultaneously without wasteful overlaps.

Common Tooling Pitfalls and How to Avoid Them

It's easy for tooling to look like the key to success however as with everything it needs to be kept in moderation. Below are some core pitfalls to lookout for :


  • Hard to find: It may be tempting to stick all your tools in one place, however tools should be stored on a context basis. This means that the tool can always be found with the associated asset or browser, for example a texture validator should be able to be opened via a right click context menu for a texture.

  • Fights the engine: A good tool shouldn't find or introduce different complexities to existing workflows, the purpose of the assistive tool is to simplify the end to end pipeline not to replace it.

  • Having multiple jobs: Keep your tools sweet and simple, as soon as a tool expands past it's initial purpose it starts becoming it's own project with it's own maintenance bugs and so on.

  • Overly Complex: This goes by the general principle of "keep it simple", tooling still requires it's own maintenance and it's own time-sink, due to a projects dependancies on it's tooling you never want to be in a position where a tools technical debt is haulting the project.

By avoiding this principles, it should allow you to plan out hassle-free tooling that allows for a project to scale in a healthy manner; make sure you are always paying attention to your toolings place in your project as it serves to assist not to hinder.
